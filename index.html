The founder story

I didn't set out to build a portfolio of projects — I set out to build systems that remove friction for the people who use them. Everything else followed from that.

Scattered inputs → one system

I started as a full-stack developer shipping client websites — corporate sites, e-commerce stores, membership platforms. Somewhere around 35 delivered projects, a pattern became impossible to ignore: almost every client was solving the same underlying problems with the wrong tools — scattered scheduling links, fragmented AI workflows, e-commerce stacks that needed five different vendors just to go live.

So I stopped building one-off sites and started building products. Chatbolt turns a single prompt into a finished task — a faster, more focused alternative to general-purpose agent platforms. Weblake is full e-commerce infrastructure that takes a store from generation to deployment to day-to-day operations, end to end. Setu-Scheduler rebuilds scheduling from the ground up — no more bolting Calendly onto everything.

Three more products are in active development right now. The thread connecting all of them is the same: take something complicated, multi-tool, and slow — and rebuild it as one fast, AI-native system.

Product-first thinking

Every build starts from the user's actual workflow, not the tech stack.
